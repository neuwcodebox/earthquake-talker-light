## 구현 순서

1. 패키지 구조와 공통 모델을 만든다.
   - 메시지 모델, 우선순위, 설정, 상태 저장소
2. 텔레그램 클라이언트를 만든다.
   - 텍스트와 사진 전송, dry-run 지원
3. 미소지진 수집원을 구현한다.
   - HTML 정규화와 변경 감지
4. 국외 지진 수집원을 구현한다.
   - XML 파싱, 국내 영향 필터, seen 상태
5. PEWS 수집원을 구현한다.
   - 바이너리 phase 파싱, 위치 JSON 보정, 그리드 이미지 생성
6. 장기 실행 루프와 CLI를 연결한다.
7. 단위 테스트를 추가한다.
   - HTML 정규화
   - 국외 지진 XML 파싱/필터
   - PEWS 비트 필드 파싱
   - 텔레그램 dry-run 또는 payload 구성
8. `uv run pytest` 및 필요한 정적 검사를 수행한다.
